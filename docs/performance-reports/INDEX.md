# Performance Reports Index

> Purpose: track how to generate and locate performance reports. Reports themselves are local/CI artifacts.

## Latest Report

Generated by `scripts/run-performance-baselines.ps1`, the latest report is stored as:

- `docs/performance-reports/latest-performance-report.md`
- `docs/performance-reports/latest-performance-report.log`

These files are not committed; they are regenerated on demand.

## How to Generate

```powershell
powershell.exe -ExecutionPolicy Bypass -File scripts/run-performance-baselines.ps1 -SkipVerifiedBuild
```

If you need a dedicated Maven repo:

```powershell
powershell.exe -ExecutionPolicy Bypass -File scripts/run-performance-baselines.ps1 -UseDedicatedRepo
```

## Report Naming

Each run also writes a timestamped report and log:

- `docs/performance-reports/performance-report-YYYYMMDD-HHMMSS.md`
- `docs/performance-reports/performance-report-YYYYMMDD-HHMMSS.log`
